[justify]Раздел «Модули» предназначен для управления разделами (модулями) сайта.
Для всех модулей доступны функции редактирования и активации/деактивации, для модуля «News (Новости)» дополнительно доступны функции:
[li] [b][i]Обновление таблиц базы[/i][/b] – обновление таблиц базы данных модуля;
[li] [b][i]Удаление таблиц базы[/i][/b] – удаление таблиц базы данных модуля;
[li] [b][i]Добавление таблиц базы[/i][/b] – добавление таблиц базы данных модуля (доступна после удаления);
[li] [b][i]Очистка таблиц базы[/i][/b] – удаляет содержимое таблиц базы данных модуля, не изменяя при этом страктуры (очистка данных).

Функции удаления и добавления таблиц базы данных модуля доступны также для модуля «Order (Форма заказа)».

[b][color=green]Редактирование данных модуля[/color][/b]
Переход к форме редактирования модуля происходит по нажатию кнопки «Редактор».
При редактировании модуля можно изменить:
[li] [b][i]Кто будет видеть[/i][/b] – устанавливаются права доступа к модулю: посетитель (все), пользователь (только авторизованные), администратор.
[li] [b][i]Для пользователей состоящих в группе[/i][/b] – можно ограничить доступ к просмотру модуля в зависимости от принадлежности пользователя к группе.
[li] [b][i]Отображение блоков в модуле[/i][/b] – можно указать, на каких местах должны отображаться блоки на страницах модуля.
[li] [b][i]Сделать видимым в блоке модулей[/i][/b] – активировать отображение ссылки на модуль в блоке block-modules.php или нет.

[color=orangered][b][i]Самостоятельное добавление модуля[/i][/b][/color]
SLAED CMS, являясь открытой системой, позволяет добавлять дополнительные модули в структуру сайта. Модули хранятся в директории: modules/ (папки новых модулей необходимо соответственно скопировать в директорию: modules/).
При помощи собственных модулей можно существенно расширять функциональность системы. В случае если дополнительный модуль содержит файл таблицы в папке модуля, к примеру: modules/news/sql/table.sql, то после добавления папки этого модуля в папку: modules/ в администраторском разделе «Модули» необходимо запустить механизм создание таблиц соответствующего модуля, нажав на значение (Добавление таблиц базы).[/justify]
