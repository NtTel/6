[justify]Раздел «Карта сайта» позволяет управлять настройками генерирования файлом sitemap.xml. Этот файл позволяет сообщать поисковым системам о том, как организовано содержание на вашем сайте. Поисковые роботы просматривают этот файл, чтобы более точно индексировать отделы и страницы.

Файлы Sitemap также можно использовать для предоставления поисковым системам метаданных о ваших страницах. Это такие сведения, как дата последних изменений, периодичность обновления контента и приоритет страницы в сравнении с другими на сайте.

Чтобы о новом файле Sitemap как можно быстрее узнали поисковые роботы, веб-мастеру необходимо внести изменения в файл robots.txt, прописав в нем особую директиву, указывающую точный маршрут к карте сайта:

Пример для протокола HTTP

[i]Sitemap: http://www.ваш_сайт/sitemap.xml[/i]

Пример для безопасного протокола HTTPS

[i]Sitemap: https://www.ваш_сайт/sitemap.xml[/i]

Данные изменения можно внести непосредственно из панели управления системой, в отделе «Редактор внедрений», закладка «Правила для роботов».

Для более быстрой индексации, рекомендуется проверить созданный файл карты сайта на наличие ошибок и добавить в поисковые системы Yandex, Google, Bing при помощи «Яндекс Вебмастер», «Google Search Console», «Bing веб-мастер».

[color=green][b]Вкладка «Главная»[/b][/color]
В данном отделе отображается информация о созданном файле карты сайта, его размер, дата создания, количество строк и непосредственно код основного файла sitemap.xml. При нажатии на кнопку «Обновить» произойдёт генерация новой карты и соответственно обновление существующего или в случае отсутствия, создание нового файла.

[color=green][b]Вкладка «Шаблон»[/b][/color]
Шаблон предназначен для оформления карты сайта, делает его более понятным и приятным для визуального просмотра обычным посетителям и не является обязательным атрибутом для поисковых систем, поэтому может быть модифицирован или полностью отключён.

При помощи механизмов стилей других типов для одного XML-документа может быть создано несколько таблиц стилей, обеспечивающих несколько окончательных сценариев отображения информации, как в нашем случае, для простого файла карты сайта и файла индекса списка карт сайта.

[u]Переменные шаблона[/u]
В шаблоне используется один тип переменных: $lan[1], $lan[2] и т.д.
Они отвечают за вывод заголовка, констант (слов в зависимости от языка выбранного посетителем, если такая возможность активирована, в противном случае на языке установленном по умолчанию) и других комплектующих в зависимости от принадлежности.

[color=green][b]Вкладка «Конфигурации»[/b][/color]
В данной вкладке устанавливаются основные настройки генерации. Пояснения по некоторым настройкам:

[li] [b][i]Модули[/i][/b] – модуля системы для которых будет генерироваться карта сайта. Для выбора двух и более вариантов удерживайте клавишу «Ctrl или Strg» в зависимости от языковой версии тастатуры. Для отключения выберите параметр «Нет».
[li] [b][i]Частота обновления[/i][/b] – параметр указывающий частоту обновления страницы. Не является обязательным атрибутом, можно отключить выбрав значение «Нет».
[li] [b][i]Приоритет[/i][/b] – указывает степень важности страницы, по сравнению с другими. Приоритет можно выставлять от 0.1 (низкий) до 1.0 (высокий). Рекомендуется расставлять следующим образом: максимальный для главной страницы 1.0, для модулей высокий 0.9, для категорий средний 0.8, для публикаций более низкий 0.7. Не является обязательным атрибутом, можно отключить выбрав значение «0».
[li] [b][i]Промежуток времени между обновлениями файла карты, в часах[/i][/b] – система может создавать карты сайта в автоматическом режиме. Промежуток между созданиями файлов карты указывается в часах.
[li] [b][i]Автоматическая генерация и обновление файла карты?[/i][/b] – генерировать файл карты или нет.
[li] [b][i]Указывать дату последнего изменения[/i][/b] – добавлять параметр даты для ссылок соответствующих отделов сайта или нет. Не является обязательным атрибутом, можно не использовать.
[li] [b][i]Генерировать ссылки[/i][/b] – выбрать отделы для которых будут генерироваться ссылки в файле карты.
[li] [b][i]Использовать шаблон оформления XSL для файла карты?[/i][/b] – использовать или нет шаблон оформления карты сайта, в случае если шаблон присутствует.
[li] [b][i]Создавать обычную структуру карты для просмотра на сайте?[/i][/b] – в случае установки данной опции, дополнительно к стандартной версии карты сайта системой будет создан файл в формате HTML для публикации и обычного просмотра в виде стандартного модуля системы. Если планируется его использование, активируйте модуль «sitemap» в панели администратора, отдел «Модули».

[hr][color=blue][b]Обратите внимание![/b][/color]
Генерация ссылок производится только для модулей активированных для просмотра на сайте «всем посетителям». Смена настроек работы и видимости модулей доступна из главной панели администратора системы, отдел «Модули».[hr]
К созданию карты сайта применяются несколько требований, о которых должен знать веб-мастер.

[li] Количество ссылок в файле Sitemap не должно превышать 50 тысяч единиц. Если их больше, то система создать несколько отдельных карт сайта, которые пропишутся в файле индекса Sitemap.
[li] Максимальный размер одного файла карты не должен превышать 10 Мб. Если это произойдёт, система автоматически уменьшит его при помощи архиватора gzip.[/justify]
