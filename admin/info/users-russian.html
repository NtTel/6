[justify]Раздел «Пользователи» предназначен для управления пользователями сайта.

[b][color=green]Управление пользователями[/color][/b]
Управление существующими пользователями происходит во вкладке «Главная».
[li] [b][i]Полное редактирование[/i][/b] – изменение информации о пользователе.
[li] [b][i]Удаление объекта[/i][/b] – удаление учетной записи пользователя.
[li] [b][i]Заблокировать IP Адрес[/i][/b] – блокировка IP-адреса пользователя. Управление заблокированными IP Адресами происходит в разделе «Безопасность» (там же можно задать сообщение, которое будет выводиться при попытке входа с заблокированного IP Адреса на сайт).

[color=orangered][b][i]Поиск пользователей[/i][/b][/color]
Справа над списком пользователей выводятся поля поиска по списку:
[li] в первом поле выбирается поле, по которому должен вестись поиск (к примеру, Ник).
[li] во втором поле указывается искомое значение.
Для запуска поиска необходимо нажать кнопку «ОК».

[hr][color=blue][b]Обратите внимание![/color][/b]
[li] При редактировании учетной записи пользователя, если установлена отметка в поле «Отправить уведомление на E-Mail пользователя», то на e-mail пользователя всякий раз при редактировании будет приходить письмо в соответствии с шаблоном.
[li] Для просмотра даты и времени регистрации и последнего входа пользователя наведите курсор мыши на иконку (i) возле нужного пользователя в первой колонке списка пользователей.[hr]
[b][color=green]Добавление пользователя[/color][/b]
Для добавления нового пользователя перейдите во вкладку «Добавить» и заполните информацию о новом пользователе.
Пояснения по некоторым полям формы добавления пользователя:
[li] [b][i]Активировать собственное меню?[/i][/b] – пользователь может сформировать свое собственное меню путем задания ссылок на пункты меню при помощи редактора.
[li] [b][i]Пункты[/i][/b] – в данное поле можно задать начальное количество пунктов. Подробнее о правилах расчета пунктов см. в Информации к разделу «Группы».
[li] [b][i]Проверка модератором сообщений пользователя?[/i][/b] – все сообщения пользователя (комментарии, темы и сообщения на форуме) будут публиковаться только после проверки модератором.
[li] [b][i]Тема оформления сайта[/i][/b] – данное поле отображается только в случае нескольких установленных тем оформления системы. Пользователь может указать, в какой теме оформления ему было бы приятней работать.
[hr][color=blue][b]Обратите внимание![/color][/b]
При добавлении пользователя обязательно необходимо указать: ник, e-mail, пароль и повтор пароля.[hr][b][color=green]Новые пользователи[/color][/b]
Во вкладке «Новые пользователи» отображается список зарегистрированных пользователей, которые не активировали свой аккаунт.
Администратор может активировать аккаунт пользователя, нажав на значение (Активировать), или удалить его (Удаление объекта).
[b][color=green]Обнулить параметры[/color][/b]
В данной вкладке, возможно обнуление пунктов, рейтингов, предупреждений и подписей у всех пользователей системы.
[hr][color=red][b][i]Внимание![/b]
Пользуйтесь данной вкладкой аккуратнее - все рейтинги пользователей (пункты) удаляются, приравниваются к нулю.[/i][/color][hr]
[b][color=green]Управление конфигурациями[/color][/b]
Во вкладке «Конфигурации» происходит настройка правил работы сайта с пользователями (настройки регистрации и аккаунта пользователей).
Рассмотрим назначение некоторых конфигурационных параметров, по которым могут возникнуть вопросы:
[li] [b][i]Тип приветствия анонимных посетителей[/i][/b] – будет ли выводиться форма входа или ссылка на регистрацию и вход ниже логотипа сайта.
[li] [b][i]Регистрация без подтверждения на E-Mail?[/i][/b] – необходимо ли пользователям подтверждать e-mail путем перехода по ссылке, высылаемой на e-mail пользователя.
[li] [b][i]Разрешить пользователям менять число публикаций?[/i][/b] – смогут ли пользователи при настройке профиля менять указанное количество публикаций на домашней странице.
[li] [b][i]Активировать регистрацию новых пользователей?[/i][/b] – смогут ли новые пользователи регистрироваться на проекта. Как правило, данная функция используется для временного отключения регистрации.
[li] [b][i]Активировать сервис uLogin для авторизации посетителей через социальные сети?[/i][/b] – позволяет пользователям авторизироваться на сайте через социальные сети, такие как: Одноклассники, Google, Яндекс, ВКонтакте, Facebook и многие другие. Используется сервис uLogin, который даёт возможность, получить единый доступ для входа на ваш сайт без дополнительной регистрации.
[li] [b][i]Подтверждение правил при регистрации пользователей?[/i][/b] – необходимо ли будет пользователю соглашаться с правилами сайта при прохождении регистрации.
[li] [b][i]Правила регистрации пользователей[/i][/b] – текст, в котором отражены правила регистрации на сайте. Например, «Соглашаясь с данными правилами я подтверждаю, что мне больше 13 лет.»
[li] [b][i]Шаблон кода сервиса uLogin для авторизации посетителей через социальные сети[/i][/b] – JavaScript код предоставляемый сервисом uLogin, в котором указаны параметры, настройки и стили блока кнопок для входа на сайт. В шаблоне кода используется переменная [url] которая в последствии автоматически заменяется системой на ваш адрес редиректа входа, после успешной авторизации.
[hr][color=blue][b]Обратите внимание![/b][/color]
[li] Атрибут: [url] является обязательным параметром, в случае его удаления авторизация на сайте работать не будет.
[li] В случае смены пароля пользователя, авторизация через социальные сети, под прежним аккаунтом, будет невозможной.
[li] Для таких пользователей так же невозможны смена и восстановление пароля при помощи стандартных процедур системы.[hr]
[li] [b][i]Запрещенные имена и E-Mail[/i][/b] – в случае, если пользователь укажет имя или e-mail из заданного списка, то система выдаст ему соответствующее предупреждение. Запрещённые имена и e-mail указываются через запятую, без пробела после запятой.[/justify]
