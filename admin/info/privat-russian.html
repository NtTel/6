[justify]Раздел «Личные сообщения» предназначен для управления и настройки личных сообщений пользователей сайта.

Администратору доступны следующие возможности:
[li] Просмотр текста сообщений;
[li] Просмотр статуса прочтения сообщений;
[li] Удаление сообщений;
[li] Настройка раздела (конфигурация).

[b][color=green]Список сообщений[/color][/b]
Постраничный список всех личных сообщений выводится в виде таблицы во вкладке «Главная». Пояснения по некоторым столбцам таблицы:
[li] [b][i]Заголовок[/i][/b] – выводится заголовок сообщения, а также иконка (i), при наведении на которую отображается полный текст сообщения.
По заголовку сообщения можно определить глубину переписки: при ответе на сообщение к теме добавляется значение «Ответ:».
[li] [b][i]Отправитель, Получатель[/i][/b] – выводится ник (имя в системе) пользователя в виде гиперссылки, переход по которой открывает страницу с информацией о профиле пользователя.
[li] [b][i]Статус[/i][/b] – если сообщение прочитано получателем, то выводится иконка (i) прочитанное сообщение. Если не прочитано – (!) непрочитанное сообщение
[li] [b][i]Функции[/i][/b] – администратор может удалить любое сообщение, нажав на значение (Удаление объекта).

[b][color=green]Настройки раздела[/color][/b]
Управление настройками сообщение происходит во вкладке «Конфигурации», где можно задать следующие настройки:
[li] [b][i]Количество публикаций на страницу пользователя[/i][/b] – заданное количество сообщений будет отображаться при просмотре пользователем раздела «Личные сообщения». Рекомендуем не делать количество комментариев на одной странице более 30, чтобы не увеличивалось время загрузки страниц.
[li] [b][i]Количество публикаций на страницу администратора[/i][/b] – заданное количество сообщений будет отображаться на одной странице раздела «Личные сообщения» в административном разделе.
[li] [b][i]Максимальное количество номеров страниц пользователей[/i][/b] – если количество страниц с комментариями превышает заданное значение, то после номера страницы, соответствующего заданному числу, будет выводиться многоточие.
[li] [b][i]Максимальное количество номеров страниц администратора[/i][/b] – аналогично предыдущей настройке, но задается для раздела управления (административный раздел).
[li] [b][i]Максимально допустимое количество знаков в слове[/i][/b] – пользователь не сможет отправить сообщение, в котором в одном из слов будет превышено указанное количество знаков (можно применять к примеру для защиты от рассылки СПАМ-гиперссылок).
[li] [b][i]Минимальный период времени между публикациями, в секундах[/i][/b] – пользователь не сможет отправить сообщение, если с момента предыдущей отправки ещё не прошло указанное количество секунд (применяется для защиты от СПАМ-рассылок через личные сообщения).
[li] [b][i]Максимальное количество входящих (сохранённых) сообщений[/i][/b] – если количество будет достигнуто, то пользователю будет предложено почистить папки с сообщениями.
[li] [b][i]Уведомлять пользователей о новых сообщениях?[/i][/b] – уведомление приходит на e-mail, указанный в профиле пользователя.
[li] [b][i]Разрешить отправлять сообщения самому себе?[/i][/b] – настройка, как правило ставится в состояние «Нет», но в некоторых случаях может пригодиться.
[li] [b][i]Выводить кнопку перехода к профилю пользователя?[/i][/b] – если указано «Да», то при просмотре сообщения пользователь будет видеть кнопку: Персональная информация, переход по которой открывает персональную информацию отправителя сообщения.
[li] [b][i]Выводить кнопку перехода на сайт пользователя?[/i][/b] – если указано «Да», то при просмотре сообщения пользователь будет видеть кнопку: Посетить сайт, переход по которой открывает сайт отправителя сообщения, если адрес сайта указан в персональной информации пользователя.
[li] [b][i]Активировать личные сообщения?[/i][/b] – если указано «Нет», то функциональность обмена личными сообщениями отсутствует.[/justify]
